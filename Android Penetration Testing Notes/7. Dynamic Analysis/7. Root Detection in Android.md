# 1. What is Rooting?

Rooting gives users privileged control (like root in Linux) over the Android OS. While useful for customization, it poses security risks (e.g., bypassing app restrictions).

# 2. Practical

## Method 1

On logging into the application a root detection error pops up, hindering further usage.
![image](https://github.com/user-attachments/assets/3fcd6dea-0a61-4f29-8b54-aeced38614f4)

One simple way to bypass root detection is using Objection:

```
objection -g com.ecardsandkeys explore
android root disable
```
![image](https://github.com/user-attachments/assets/d3127d58-20f9-4314-b63e-c78e54c1f18a)

Sometimes it won’t work as expected, for this we have other alternatives too.

## Method 2

For this method, reverse engineer the application using jadx-gui. Open the application in jadx-gui.

The root prompt appears when clicking the login button, so explore the login activity in jadx-gui.
![image](https://github.com/user-attachments/assets/41a936cd-3ded-4d39-9b93-67d92391adb8)


Use search in jadx-gui to find “root”.

On expanding the login activity, find the `checkRootDevice` method.
![image](https://github.com/user-attachments/assets/928ddb80-224f-4d2f-844c-eecfdaf0760f)

![image](https://github.com/user-attachments/assets/fc4f8a99-ce30-449d-ab9c-d96fdb8289c3)



List all methods using Objection:

```
objection -g com.ecardsandkeys explore
android hooking list class_methods com.ecardsandkeys.activity.LoginActivity
```
![image](https://github.com/user-attachments/assets/c90d45dd-adf7-4595-9092-1cfaee75caa3)


Change the boolean return value of `checkRootedDevice`:

```
android hooking set return_value com.ecardsandkeys.activity.LoginActivity.checkRootedDevice false
```
![image](https://github.com/user-attachments/assets/7258bbb4-bdfc-4e89-80bd-5887525db72c)


This bypasses root detection and allows login.
